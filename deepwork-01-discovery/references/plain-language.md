# Plain-Language Mandate — Klartext-Pflicht

See `deepwork-00-bootstrap/references/plain-language.md` for the full mandate.

Key rules:
- All user-facing output: plain language first, technical details second
- Status updates during work: always in Klartext (no technical tool names visible)
- Pre-Permission-Block before every confirmable step

- End-of-Phase-Reports: lead with "What did we achieve and what does it mean for you?"
