# Concept Protocols — deepwork-03-execute Context

Fuer project_type: concept, research, marketing

Vollstaendige Protokoll-Definitionen: `deepwork-env-setup/references/concept-protocols.md`

## Empfohlene Ausfuehrungs-Reihenfolge

1. **StructuredPlanning** → Problemstellung + Bewertungsmatrix + Empfehlung + Annahmen
2. **StakeholderMap** → Alle Quadranten + Interessen-Konflikte + Kommunikationsplan
3. **ScenarioAnalysis** → 3 Szenarien + Fruehindikatoren + Review-Termin
4. **RoadmapBuild** → ICE-Score-Backlog + phasierter Plan + Abhaengigkeiten
5. **RiskMatrix** → Top-5-Risiken + Massnahmen + Owner + Heatmap
6. **PitchDeck** → 10-Slide-Struktur + Anti-Pattern-Check + Storytelling-Test

## Verification-Kriterien fuer Concept-Sections

- StructuredPlan: Empfehlung durch Bewertungsmatrix belegt? Annahmen dokumentiert?
- StakeholderMap: Alle 4 Quadranten gefuellt? Konflikte adressiert? Kommunikationsplan konkret?
- Roadmap: Abhaengigkeiten kartiert? Kapazitaet gegenueber gestellt? Realistisch?
- RiskMatrix: Owner je Top-Risiko benannt? Fruehwarnsignale konkret?
- PitchDeck: Zahlen verifiziert? Storytelling-Test bestanden? Anti-Patterns eliminiert?
