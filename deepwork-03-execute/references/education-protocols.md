# Education Protocols — deepwork-03-execute Context

Fuer project_type: education, concept, content

Vollstaendige Protokoll-Definitionen: `deepwork-env-setup/references/education-protocols.md`

## Empfohlene Ausfuehrungs-Reihenfolge

1. **LearnerPersona** → Persona-Karte(n) mit Vorwissen + Constraints + Lernstil
2. **LearningOutcomes** → LZ-Set mit Blooms-Level + Pruefbarkeit + Constructive Alignment
3. **CurriculumOutline** → Wissens-Graph + Modul-Uebersicht + Zeitschaetzungen
4. **ConceptExplanation** → Pro Kernkonzept: Analogie + Detail + Anwendung + Selbsttest
5. **ExerciseDesign** → Pro Lernziel: Aufgabe + Musterloesung + Fehler-Feedback + Rubrik
6. **AssessmentDesign** → Formative Checks pro Modul + summative Abschlusspruefung

## Verification-Kriterien fuer Education-Sections

- Jedes Lernziel durch mindestens eine Uebung und eine Assessment-Frage abgedeckt?
- Constructive Alignment: Lernziel = Lernaktivitaet = Pruefung?
- Jede Aufgabe: Musterloesung vorhanden? Typische Fehler antizipiert?
- Assessment: Schwierigkeitsverteilung 30/50/20 eingehalten?
- Feedback-Nachrichten lehrreich, nicht beschaemend?

## Qualitaets-Prinzip

Lernende sind Experten ihrer eigenen Lernerfahrung. Haerte Tests mit der echten Zielgruppe (oder einer stellvertretenden Persona) bevor Inhalte als fertig gelten.
