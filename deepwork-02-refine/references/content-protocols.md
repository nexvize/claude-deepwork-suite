# Content Protocols — Referenz (deepwork-02-refine)

In der Refine-Phase für Content-Projekte:

Sections entsprechen typischerweise einzelnen Artikeln, Kapiteln oder Content-Stücken.
Jede Section hat einen eigenen Outline → Draft → Edit Zyklus.

Sektion-Definitionen für Content-Projekte:
- Section = ein abgeschlossenes Content-Stück (Artikel, Kapitel, Landing-Page-Block)
- Vertical-Slice-Check: Ist die Section alleine veröffentlichbar?
- Dependencies: Welche Sections bauen auf anderen auf (Verlinkungen, Referenzen)?
