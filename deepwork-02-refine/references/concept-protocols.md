## Section-Splitting fuer Concept-Projekte

| Section-Typ | Protokoll | Output |
|---|---|---|
| Problemdefinition + Optionen | StructuredPlanning | Bewertungsmatrix + Empfehlung |
| Stakeholder-Analyse | StakeholderMap | Stakeholder-Liste + Kommunikationsplan |
| Zukunftsszenarien | ScenarioAnalysis | 3 Szenarien + Fruehindikatoren |
| Pitch-Erstellung | PitchDeck | 10-Slide-Struktur + Anti-Patterns |
| Umsetzungsplanung | RoadmapBuild | ICE-Score-Backlog + phasierter Plan |
| Risikobewertung | RiskMatrix | Top-5-Risiken + Massnahmen + Owner |

**Vertical Slice:** Jede Section liefert eigenstaendige, lieferbare Dokumente.

**Reihenfolge-Empfehlung:** StructuredPlanning zuerst (definiert Richtung), dann StakeholderMap + RiskMatrix (parallel), dann RoadmapBuild und PitchDeck.
