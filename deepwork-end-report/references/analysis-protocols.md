# Analysis Protocols — deepwork-end-report Context

Fuer project_type: analysis, research, finance

Vollstaendige Protokoll-Definitionen: `deepwork-env-setup/references/analysis-protocols.md`

## Typ-spezifische Abschluss-Checks

**ReportDraft Quality Check:**
- Executive Summary ohne Fachkenntnisse verstaendlich?
- Jede Empfehlung mit Owner + Deadline + Impact?
- Appendix vollstaendig (Methodologie, Datenquellen, Glossar)?
- Alle Zahlen intern konsistent?

**CompetitiveAnalysis (falls durchgefuehrt):**
- Monitoring-Plan fuer Wettbewerber definiert?
- Review-Termin gesetzt (Quartalsweise empfohlen)?

**DashboardSpec (falls durchgefuehrt):**
- Alert-Schwellenwerte definiert?
- Refresh-Rate und Zugriffsrechte spezifiziert?
- Verantwortlicher fuer Dashboard-Pflege definiert?

## Abschluss-Dateien (analysis-spezifisch)

`PROJECT-COMPLETE.md`
- Wichtigste Insights + Quantifizierungen
- Datenquellen-Uebersicht + Qualitaetsbewertung
- Naechste Review-Termine (Dashboard, Competitive)

`NEXT-STEPS.md`
- Empfehlungen aus ReportDraft mit Prioritaeten + Owner
- Offene Fragen aus InsightExtraction

`KNOWLEDGE-BASE.md`
- Schema-Dokumentation fuer Wiederverwendung
- Wiederverwendbare Analyse-Vorlagen
- Benchmark-Werte fuer kuenftige Vergleiche
