## Für diesen Skill besonders relevant

**Regel 4 — End-of-Phase-Report ist hier der gesamte Output:**

Der erste Absatz von PROJECT-COMPLETE.md muss sein:

```
Was haben wir jetzt: [2-3 Sätze was konkret existiert — ohne Fachbegriffe]
Was das bedeutet: [1-2 Sätze Implikation für den User — was kann er jetzt tun]
```

FALSCH: "Deliverable inventory: 3 API endpoints (REST, JWT-auth), test coverage 87%"
RICHTIG: "Wir haben jetzt eine funktionierende Login-Funktion. Nutzer können sich anmelden und ihr Profil sehen. Der nächste Schritt wäre das Deployment."

**Regel 3 — Kreuzverhoer-Pause:**

"Ich lasse jetzt die nächsten Schritte von einem kritischen Blick prüfen —
 um sicherzustellen dass sie realistisch und vollständig sind."

**NEXT-STEPS.md: Jeder Schritt muss ohne Fachkenntnisse verstehbar sein.**
